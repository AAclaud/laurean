1. El sistema Laurean

Laurean es una tienda de moda elegante y práctica. El sistema tiene tres partes que comparten la misma marca:

Tienda pública
Lo que ve el cliente: portada, colecciones (Women / Men / Kids), catálogo, producto, carrito y pago. Páginas: Laurean.html, laurean-women/men/kids.html, catalogo.html, producto.html, nosotros.html.
Panel admin
El "cerebro": pedidos, productos, inventario, vendedores, comisiones, configuración. Página: admin.html.
Punto de venta (POS)
Para vender en tienda física, rápido. Página: pos.html.
Identidad
"El arte de vestir bien". Submarcas Women (rubí), Men (gunmetal) y Kids (celeste). Todo comparte el mismo logotipo y voz.
Local-first: el sistema guarda todo primero en el navegador y, si hay conexión a la base de datos (Supabase), sincroniza en la nube. Para la demo funciona aunque no haya nube.

2. Roles y accesos

RolQué puede hacer
SuperusuarioTodo. Único que ve la "Zona de Peligro" (borrar pedidos, restaurar precios) y crea administradores.
AdministradorGestiona catálogo, pedidos, vendedores, inventario, comisiones, Configuración General, Canales y Pagos.
VendedorTiene su código de referido y ve sus precios/comisiones. Puede usar el POS si se le habilita.
BodegaInventario por bodega; precios con descuento de bodega.
Agente de pedidosSolo ve y gestiona pedidos.
Los precios cambian según quién esté logueado: público, vendedor (con su % menos) o bodega. Un indicador flotante ("Precios de administrador / vendedor") lo recuerda en la tienda.

3. Cómo ingresar

  1. Abre login.html (o el botón de cuenta 👤 en la tienda).
  2. Escribe correo y contraseña del usuario.
  3. El sistema te lleva a tu panel según tu rol (admin → dashboard, vendedor → su panel, etc.).

Para volver a la tienda, usa "Volver a la tienda". Para salir, "Cerrar sesión".

4. La tienda (lo que ve el cliente)

Portada

  • Selector de categorías: Mujer / Hombre / Laurean Kids / Ofertas. Al elegir una, la pantalla se ambienta con el color de esa submarca y muestra un breve "splash" con su logo.
  • Las colecciones: 3 tarjetas que llevan a las páginas Women / Men / Kids.
  • Productos: cada tarjeta tiene "Agregar al carrito". Los combos se agregan como bloque (no se pueden romper para abusar del precio).
  • Programa de vendedores: banda que invita a unirse (lleva a vendedoras.html).

Carrito y pago

  1. El cliente agrega productos y abre el carrito (🛍️).
  2. Ajusta cantidades o quita productos. Los combos se quitan completos con "Quitar combo".
  3. Llena sus datos y elige método de pago.
  4. El pedido entra al panel admin → Pedidos.

6. Tareas paso a paso

Publicar un producto al catálogo

  1. Admin → Productos → elige el producto (o créalo).
  2. Abre "Publicar al catálogo": pon el precio público, marca "Mostrar precio sin login" si quieres que se vea, elige categoría y sube foto.
  3. Guarda. Aparece en la tienda en su categoría.
Tip de fotos: usa imágenes donde el sujeto se vea completo (sin cortar la cabeza). El sistema recorta para llenar el espacio, así que las fotos verticales y centradas se ven mejor.

Crear un vendedor

  1. Admin → Vendedores → "Nuevo usuario".
  2. Nombre, correo, contraseña (mín. 6), teléfono y rol "Vendedor".
  3. Opcional: % de comisión propio, cumpleaños, acceso a POS.
  4. Guardar. Se genera su código de referido (LAU-XXXX).

Agregar un canal (redes / contacto del footer)

  1. Admin → Configuración → "Canales".
  2. "+ Agregar canal", elige el icono (Instagram, WhatsApp, Email, Teléfono…), escribe la etiqueta y el enlace, marca "Visible".
  3. "Guardar contenido". Aparece en el footer de todas las páginas.

Editar anuncios y WhatsApp

Admin → Configuración → "Contenido del sitio": edita los mensajes del marquee de la portada y el número de WhatsApp.

7. Punto de venta (POS)

pos.html — para vender en tienda física. Se agregan productos al ticket, se cobra y se cierra la venta. Solo los usuarios con "acceso a POS" habilitado pueden entrar.

8. Pagos

Para la demo, los pagos se manejan de forma manual/asistida (transferencia, QR, link o efectivo). El cobro con tarjeta en línea (QPayPro) está programado y se activa cuando el comercio esté activo y se carguen las llaves.

Pasarela QPayPro: requiere que el comercio esté activo (suscripción vigente). Si aparece inactivo, es trámite con QPayPro (ver sección 10).

9. Preguntas frecuentes

¿Por qué los precios cambian?

Porque dependen de quién está logueado (público, vendedor, bodega). Es normal.

La sección de Pagos dice "no hay transacciones"

Es normal si aún no se ha conectado/creado la base de datos de pagos. No es un error de uso.

¿Puedo trabajar sin internet?

Sí, para la demo y operación básica: el sistema guarda en el navegador. La nube es para sincronizar entre dispositivos y respaldar.

10. Pendientes para producción (técnico)

Para pasar de la demo local al sitio en vivo hace falta:

  • Generar js/config.js con las llaves de Supabase (URL + anon) en el build.
  • Aplicar supabase/schema.sql en el proyecto Supabase (crea las tablas: pedidos, pagos, perfiles, inventario…).
  • Desplegar las Edge Functions: forza-proxy, qpaypro-proxy y create-user, con sus secretos.
  • Crear el primer usuario superusuario y el bucket de imágenes product-images.
  • Activar el comercio de la pasarela de pagos (QPayPro) y cargar sus llaves.
  • Desplegar en Vercel con las variables de entorno y el dominio.
