Integración Forza Delivery
+ análisis de costos

Esta documentación describe la integración entre la tienda Laurean y la API de Forza Delivery Express vía Supabase Edge Functions, y presenta el análisis de costos mensuales para operar el stack completo.

Resumen ejecutivo

Forza Delivery Express es el courier que se utilizará para todos los envíos nacionales en Guatemala. Su API (documentación oficial) permite automatizar toda la operación logística: cotizar, generar guías, rastrear, recolectar y cancelar.

La integración está construida sobre tres capas:

  1. Frontend (envios.html): UI para operar todo el ciclo de envío.
  2. Edge Function (forza-proxy en Supabase, Deno): firma HMAC-SHA256 con la SecretKey de Forza, valida sesión del admin, llama a Forza.
  3. Webhook (forza-webhook): recibe eventos automáticos de cambio de estado y actualiza la tabla orders en Supabase.
¿Por qué Edge Function y no llamar directo desde el navegador?
Forza autentica con HMAC-SHA256 firmado con una SecretKey. Si esa clave vivera en el navegador, cualquier persona podría inspeccionarla y generar guías a tu nombre. La Edge Function la mantiene del lado servidor.

El frontend nunca toca a Forza directamente. Toda llamada pasa por la Edge Function que:

  • Verifica el JWT del usuario contra Supabase Auth.
  • Confirma que el rol sea admin o superuser.
  • Inyecta credenciales (CodApp, CodeOfReference, IdClient).
  • Firma el body con HMAC-SHA256 → header LauValue (44 chars base64).
  • Hace POST al endpoint correspondiente.
  • Devuelve la respuesta tal cual.

Ejemplo: cotizar tarifa

const r = await Forza.quoteRate({
  HeaderCodeSource:    '0101',    // Guatemala / Ciudad
  HeaderCodeDestiny:   '0301',    // Sacatepéquez / Antigua
  Weight:              2.5,        // kg
  Dateofsale:          '2026-05-25 14:30:00',
  TypeServiceDelivery: 'STD',     // 'COD' para contra entrega
});
if (r.ok) {
  console.log('Tarifa Q', r.data.Response.Rate);
}

Ejemplo: generar guía COD

const r = await Forza.createGuide({

});

Flujo de pedido completo

  1. Cliente compra en Laurean.html y confirma checkout.
  2. Frontend hace INSERT en tabla orders de Supabase con datos del cliente y items.
  3. (Opcional, si está activado auto-Forza) frontend llama Forza.createGuide() y guarda forza_guide_serie/forza_guide_number en el pedido.
  4. Admin abre envios.html → tab "Crear guía" si quiere generar manualmente.
  5. Forza recoge el paquete → emite evento PUP → forza-webhook actualiza estado a en_preparacion.
  6. Eventos siguientes (OFD = en ruta, DLV = entregado) actualizan automáticamente orders.status.
  7. Si es COD: Forza cobra al destinatario y deposita a la cuenta vinculada (Forza maneja la liquidación).

Setup paso a paso

  1. Crear proyecto Supabase en app.supabase.com (Free tier).
  2. Correr el schema: SQL Editor → pegar y ejecutar supabase/schema.sql.
  3. Correr el seed: SQL Editor → pegar y ejecutar supabase/seed.sql (categorías base).
  4. Crear usuarios admin en Authentication → Users → Add user (manual). Después correr el UPSERT en seed.sql con sus UUIDs.
  5. Configurar el frontend: copia js/config.example.js → js/config.js y pega SUPABASE_URL + SUPABASE_ANON_KEY de tu proyecto.
  6. Instalar Supabase CLI: brew install supabase/tap/supabase
  7. Login y link: 
    supabase login
supabase link --project-ref <tu-project-ref>
  8. Configurar secrets de Forza: 
    supabase secrets set \
  FORZA_BASE_URL=https://sandbox.apicore.forzadelivery.io:40467 \
  FORZA_COD_APP= \
  FORZA_SECRET_KEY= \
  FORZA_CODE_OF_REFERENCE= \
  FORZA_ID_CLIENT=
  9. Deploy de las Edge Functions: 
    supabase functions deploy forza-proxy
supabase functions deploy forza-webhook
  10. Registrar webhook con Forza: contactar al asesor de Forza con la URL https://<tu-proyecto>.supabase.co/functions/v1/forza-webhook y pedir que activen las notificaciones.
  11. Probar: abrir envios.html, ir a "Cotizar tarifa" y verificar que devuelve un costo.

FAQ operativo

¿Cómo cancelo una guía ya creada?

Sólo se pueden cancelar guías que aún no han sido recolectadas por Forza. En envios.html → tab "Mis guías" → botón "Cancelar" sobre la fila correspondiente. Internamente llama a SetCancelGuides. Después de cancelar, el estado en orders pasa a cancelado automáticamente.

¿Qué hago si el cliente no contesta cuando llega el repartidor?

Forza intenta entregar hasta 3 veces. Después de la tercera, regresa el paquete al remitente y emite el evento RTN (devolución). Aparecerá en orders.forza_tracking_status como "Devuelto". Coordina con el cliente y genera una nueva guía manualmente.

¿COD me deposita automáticamente?

Sí. Forza tiene un ciclo de liquidación (consulta con tu asesor — típicamente semanal o quincenal). El monto se deposita a la cuenta bancaria vinculada en tu contrato. La integración API solo automatiza la generación de la guía; la liquidación financiera es offline.

¿Cómo sé si la Edge Function falla?

Supabase Dashboard → Functions → forza-proxy → tab "Logs". Cualquier error queda registrado. Para alertas automáticas (Slack/email), se puede agregar un webhook desde Supabase o usar UptimeRobot (gratis hasta 50 monitores).

¿Puedo usar otro courier además de Forza?

Sí. La arquitectura es genérica: basta crear otra Edge Function (cargo-expreso-proxy, etc.) y otro wrapper en js/. El frontend puede ofrecer al admin elegir courier por pedido.

¿La SecretKey está realmente segura?

Sí. Vive sólo como variable de entorno en Supabase Edge Functions (encriptada at-rest). Nunca se serializa al cliente. La verificación en cada request:

  1. Browser envía JWT del usuario admin.
  2. Edge Function valida el JWT y consulta profiles para confirmar rol.
  3. Sólo entonces firma con la SecretKey y llama a Forza.
